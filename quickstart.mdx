This guide will walk you through creating a simple bot with a ping command. By the end, you'll have a working bot that responds to messages.

## Prerequisites

Before you begin, make sure you have:

- Python 3.10 or higher installed
- fluxer.py installed (`pip install fluxer.py`)
- A Fluxer bot token (see below)

## Getting a Bot Token

<Steps>
  <Step title="Create a bot application">
    1. Go to the [Fluxer Developer Portal](https://fluxer.app/developers)
    2. Click "New Application"
    3. Give your application a name
    4. Navigate to the "Bot" section
  </Step>

  <Step title="Generate a token">
    Click "Reset Token" to generate a new bot token.

    <Warning>
      Keep your bot token secret! Anyone with your token can control your bot. Never commit tokens to version control or share them publicly.
    </Warning>

    Copy the token somewhere safe - you'll need it in the next step.
  </Step>

  <Step title="Invite your bot">
    Use the OAuth2 URL generator in the developer portal to create an invite link for your bot, then add it to a server where you can test it.
  </Step>
</Steps>

## Your First Bot

Let's create a simple bot that responds to a `!ping` command.

<Steps>
  <Step title="Create a Python file">
    Create a new file called `bot.py` and open it in your favorite editor.
  </Step>

  <Step title="Write the bot code">
    Add the following code to `bot.py`:

    ```python bot.py
    import fluxer

    bot = fluxer.Bot(command_prefix="!", intents=fluxer.Intents.default())

    @bot.event
    async def on_ready():
        print(f"Bot is ready! Logged in as {bot.user.username}")

    @bot.command()
    async def ping(ctx):
        await ctx.reply("Pong!")

    if __name__ == "__main__":
        TOKEN = "your_bot_token"
        bot.run(TOKEN)
    ```

    Let's break down what this code does:

    - **Line 1**: Import the fluxer library
    - **Line 3**: Create a bot instance with `!` as the command prefix and default intents
    - **Lines 5-7**: Define an event handler that runs when the bot is ready
    - **Lines 9-11**: Define a command called `ping` that replies with "Pong!"
    - **Lines 13-15**: Run the bot with your token
  </Step>

  <Step title="Add your token">
    Replace `"your_bot_token"` with the token you copied from the developer portal.

    <Info>
      For production bots, use environment variables instead of hardcoding tokens:
      ```python
      import os
      TOKEN = os.getenv("FLUXER_TOKEN")
      ```
    </Info>
  </Step>

  <Step title="Run your bot">
    Run the bot from your terminal:

    ```bash
    python bot.py
    ```

    You should see the "Bot is ready!" message in your console when the bot successfully connects.
  </Step>

  <Step title="Test the command">
    In any channel your bot has access to, send:

    ```
    !ping
    ```

    Your bot should reply with "Pong!"

    Congratulations! You've created your first Fluxer bot!
  </Step>
</Steps>

## Understanding the Code

### The Bot Instance

```python
bot = fluxer.Bot(command_prefix="!", intents=fluxer.Intents.default())
```

The `Bot` class is the high-level interface for creating bots with a command framework. Key parameters:

- `command_prefix`: The string that users type before command names (e.g., `!`, `?`, `/`)
- `intents`: Controls which events your bot receives from the gateway

<Note>
  `Intents.default()` provides a reasonable set of permissions for most bots. For more control, see the Intents documentation.
</Note>

### Events

```python
@bot.event
async def on_ready():
    print(f"Bot is ready! Logged in as {bot.user.username}")
```

Events are triggered when something happens in Fluxer (a message is sent, a user joins, etc.). The `@bot.event` decorator registers an event handler. The `on_ready` event fires when the bot successfully connects to the gateway.

### Commands

```python
@bot.command()
async def ping(ctx):
    await ctx.reply("Pong!")
```

Commands are functions decorated with `@bot.command()`. The function name becomes the command name. The `ctx` parameter is a context object containing information about the command invocation:

- `ctx.message` - The message that triggered the command
- `ctx.author` - The user who sent the command
- `ctx.channel` - The channel where the command was sent
- `ctx.reply()` - Send a reply to the command

## Common Patterns

### Adding Command Arguments

Commands can accept arguments:

```python
@bot.command()
async def say(ctx, *, text: str):
    await ctx.send(text)
```

Usage: `!say Hello, world!`

### Handling Message Events

You can respond to all messages, not just commands:

```python
@bot.event
async def on_message(message):
    # Ignore messages from the bot itself
    if message.author.id == bot.user.id:
        return
    
    # Process commands
    await bot.process_commands(message)
```

<Warning>
  If you implement `on_message`, you must call `bot.process_commands(message)` manually, or commands won't work!
</Warning>

### Using Different Intents

For access to all events:

```python
bot = fluxer.Bot(command_prefix="!", intents=fluxer.Intents.all())
```

For specific intents:

```python
intents = fluxer.Intents.default()
intents.message_content = True
intents.guild_members = True
bot = fluxer.Bot(command_prefix="!", intents=intents)
```
