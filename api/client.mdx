## Overview

The `Client` class is the low-level client that connects to Fluxer and dispatches events. This gives you full control over the gateway lifecycle. For most bots, use the `Bot` subclass instead.

## Constructor

```python
from fluxer import Client, Intents

client = Client(
    intents=Intents.default(),
    api_url=None,
    max_retries=5,
    retry_forever=False
)
```

### Parameters

<ParamField path="intents" type="Intents" default="Intents.default()">
  Gateway intents to request. Controls which events your bot receives.
</ParamField>

<ParamField path="api_url" type="str | None" default="None">
  Base URL for the Fluxer API (default: https://api.fluxer.app/v1). Use this to connect to self-hosted Fluxer instances.
</ParamField>

<ParamField path="max_retries" type="int" default="5">
  Maximum number of retries for HTTP requests.
</ParamField>

<ParamField path="retry_forever" type="bool" default="False">
  Whether to retry HTTP requests indefinitely.
</ParamField>

## Properties

### user

```python
client.user -> User | None
```

The bot user, available after the READY event.

### guilds

```python
client.guilds -> list[Guild]
```

List of guilds the bot is in (populated from READY + GUILD_CREATE events).

## Event Registration

### event()

```python
@client.event
async def on_message(message):
    ...
```

Decorator to register an event handler. The function name determines the event.

**Supported events:**
- `on_ready` → READY
- `on_message` → MESSAGE_CREATE
- `on_message_edit` → MESSAGE_UPDATE
- `on_message_delete` → MESSAGE_DELETE
- `on_guild_join` → GUILD_CREATE
- `on_guild_remove` → GUILD_DELETE
- `on_member_join` → GUILD_MEMBER_ADD
- `on_member_remove` → GUILD_MEMBER_REMOVE
- `on_channel_create` → CHANNEL_CREATE
- `on_channel_update` → CHANNEL_UPDATE
- `on_channel_delete` → CHANNEL_DELETE
- `on_voice_state_update` → VOICE_STATE_UPDATE
- `on_resumed` → RESUMED
- `on_raw_reaction_add` → MESSAGE_REACTION_ADD (raw)
- `on_raw_reaction_remove` → MESSAGE_REACTION_REMOVE (raw)
- `on_raw_reaction_clear` → MESSAGE_REACTION_REMOVE_ALL (raw)
- `on_raw_reaction_clear_emoji` → MESSAGE_REACTION_REMOVE_EMOJI (raw)

### on()

```python
@client.on("message")
async def handle_msg(message):
    ...
```

Decorator to register an event handler with an explicit name.

<ParamField path="event_name" type="str" required>
  The event name (without "on_" prefix)
</ParamField>

## HTTP Methods

### fetch_channel()

```python
await client.fetch_channel(channel_id)
```

Fetch a channel from the API (not cache).

<ParamField path="channel_id" type="str" required>
  The channel ID to fetch
</ParamField>

<ResponseField name="return" type="Channel">
  The channel object
</ResponseField>

### fetch_message()

```python

```

Fetch a message from the API by channel ID and message ID.

<ParamField path="channel_id" type="str" required>
  The channel ID
</ParamField>

<ParamField path="message_id" type="str" required>

</ParamField>

  The message object
</ResponseField>

## Connection Lifecycle

### start()

```python
await client.start(token)
```

Connect to Fluxer and start receiving events (async version). Use this if you're managing your own event loop.

<ParamField path="token" type="str" required>
  Your bot token
</ParamField>

### close()

```python
await client.close()
```

Disconnect from the gateway and clean up resources.

### run()

```python
client.run(token)
```

Blocking call that connects to Fluxer and runs the bot. This is the simplest way to start your bot. It creates an event loop, calls start(), and handles cleanup.

<ParamField path="token" type="str" required>
  Your bot token
</ParamField>

## Example Usage

```python
from fluxer import Client, Intents

client = Client(intents=Intents.default())

@client.event
async def on_ready():
    print(f"Logged in as {client.user.username}")

@client.event
async def on_message(message):
    if message.content == "!ping":
        await message.reply("Pong!")

client.run("your_token_here")
```
