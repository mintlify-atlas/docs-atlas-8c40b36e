## Overview

Permissions control what actions users can perform in a guild or channel. Each permission is a bit flag that can be combined using bitwise operators.

## Usage

```python
from fluxer import Permissions

# Combine multiple permissions
mod_perms = Permissions.KICK_MEMBERS | Permissions.BAN_MEMBERS | Permissions.MANAGE_MESSAGES

# Check if a permission is set
if role.permissions & Permissions.ADMINISTRATOR:
    print("User is an admin!")

# Check multiple permissions
required = Permissions.SEND_MESSAGES | Permissions.EMBED_LINKS
if (user_perms & required) == required:
    print("User can send messages with embeds")
```

## Examples

### Check Admin Permission

```python
from fluxer import Permissions

@bot.event
async def on_message(message):
    if message.guild_id:
        member = await bot.get_guild_member(message.guild_id, message.author.id)
        if member.permissions & Permissions.ADMINISTRATOR:
            print(f"{message.author} is an administrator!")
```

### Require Multiple Permissions

```python
from fluxer import Permissions
import fluxer.checks

@bot.command()
@fluxer.checks.has_permission(Permissions.KICK_MEMBERS | Permissions.BAN_MEMBERS)
async def moderate(ctx):
    await ctx.reply("You can both kick and ban members!")
```

### Create Role with Permissions

```python
from fluxer import Permissions

# Moderator permissions
mod_perms = (
    Permissions.KICK_MEMBERS |
    Permissions.BAN_MEMBERS |
    Permissions.MANAGE_MESSAGES |
    Permissions.VIEW_AUDIT_LOG
)

role = await guild.create_role(
    name="Moderator",
    permissions=int(mod_perms),
    color=0x00FF00
)
```

### Check Text Channel Permissions

```python
from fluxer import Permissions

required_perms = (
    Permissions.VIEW_CHANNEL |
    Permissions.SEND_MESSAGES |
    Permissions.EMBED_LINKS
)

if (user_perms & required_perms) == required_perms:
    await channel.send("You have all required permissions!")
else:
    print("Missing permissions")
```

## Permission Hierarchy

1. **Administrator** - Bypasses all permission checks
2. **Guild Owner** - Has all permissions automatically
3. **Role Permissions** - Cumulative from all roles the user has
4. **Channel Overwrites** - Can allow or deny specific permissions in channels

## Notes

- Permissions are stored as a 64-bit integer
- Use bitwise OR (`|`) to combine permissions
- Use bitwise AND (`&`) to check if a permission is present
- The `ADMINISTRATOR` permission grants all other permissions
- Guild owners bypass all permission checks
